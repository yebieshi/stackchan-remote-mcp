## v0.1.1 连续拍照修复验证

曾复现连续拍照超时：设备日志已经显示 `frame acquired` 和
`uploaded ... -> relay OK`，但 MCP 仍把新照片判断为旧照片。根因在
Relay / MCP 的新旧图片判断，不在摄像头取帧。

修复包括：

- 每次上传生成单调递增的 `X-Photo-Version`
- 使用纳秒级文件修改时间补充元数据
- 请求加入缓存规避参数，并用 SHA-256 比较图片内容
- 把等待新图片的轮询窗口延长到 15 秒
- 摄像头取帧增加短暂重试，同时保留预览子窗口

修复后已连续三次远程拍照，并成功返回三张不同的新图片。
